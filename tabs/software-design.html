Esta sección detalla el modelo de dominio, el diagrama de clases, y otras visualizaciones estructurales y de comportamiento que sustentan la arquitectura lógica del sistema. Estas representaciones permiten comprender cómo interactúan las entidades clave dentro de la aplicación financiera diseñada para PYMES.

📘 Modelo del Dominio

Este diagrama muestra una visión general de las entidades principales del sistema, como Empresa, Usuario, Cliente, Factura, entre otras. Representa cómo se relacionan entre sí, ayudando a visualizar la estructura conceptual completa.

🧱 Diagrama de Clases

Aquí se describen las propiedades y métodos de cada clase. Este diagrama es esencial para el diseño orientado a objetos, mostrando herencias, asociaciones y comportamientos clave del sistema.

🧍 Diagramas de Objetos

Los diagramas de objetos representan ejemplos reales e instanciados de entidades del sistema. Son útiles para entender cómo se verían los datos cargados en el sistema durante su uso.

🏢 Empresa y Usuario

Este diagrama ejemplifica una empresa real y su relación con un usuario con rol de administrador.

🧾 Factura, Cliente y Proyecto

Ejemplo de una factura emitida, asociada a un cliente y a un proyecto específico.

📊 Reporte Mensual

Visualiza cómo un reporte financiero puede estar vinculado a una empresa.

📈 Diagramas de Estados

Estos diagramas muestran el ciclo de vida de las entidades más relevantes. Representan los distintos estados que puede tomar un objeto según acciones del sistema o del usuario.

👤 Usuario

Desde el registro hasta su posible eliminación, pasando por activación o suspensión.

🏗️ Proyecto

Evolución de un proyecto desde su planificación hasta su finalización o cancelación.

🧾 Factura

Transición típica de una factura desde su estado en borrador hasta ser pagada, anulada o vencida.
